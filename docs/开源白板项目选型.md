## 详细对比

### 1. Excalidraw

**简介**：虚拟白板，用于绘制手绘风格的图表

**GitHub**: https://github.com/excalidraw/excalidraw

**核心特点**：
- ✅ **手绘风格** - 独特的手绘美学，亲切自然
- ✅ **极简设计** - 低学习成本，专注核心功能
- ✅ **端到端加密** - 安全性强
- ✅ **社区活跃** - 11 万+ Stars，持续维护
- ✅ **MIT 协议** - 商业友好

**技术栈**：
- React + TypeScript
- Rough.js（手绘风格渲染）
- 自建 Canvas 引擎

**适用场景**：
- 流程图、架构图
- 团队协作讨论
- 教学演示
- 快速原型草图

**优势**：
- 🟢 用户体验极佳，学习成本低
- 🟢 社区最活跃，生态最完善
- 🟢 MIT 协议，二次开发友好
- 🟢 手绘风格独特，差异化明显

**劣势**：
- 🔴 功能相对简单，不支持复杂图形
- 🔴 自建引擎，扩展性相对弱
- 🔴 主要面向图表，不是通用白板

**官网**: https://excalidraw.com/

---

### 2. tldraw

**简介**：无限画布 SDK，开发者友好的白板解决方案

**GitHub**: https://github.com/tldraw/tldraw

**核心特点**：
- ✅ **SDK 定位** - 专为开发者设计，高度可定制
- ✅ **无限画布** - 支持无限缩放和移动
- ✅ **插件架构** - 易于扩展和集成
- ✅ **实时协作** - 内置 multiplayer 支持
- ✅ **性能优秀** - 渲染效率高

**技术栈**：
- React + TypeScript
- Signia（响应式状态管理）
- 自建渲染引擎

**适用场景**：
- 作为 SDK 集成到产品中
- 需要深度定制的白板功能
- 多人实时协作场景
- 构建类似 Figma 的产品

**优势**：
- 🟢 **最适合二次开发** - SDK 架构，扩展性强
- 🟢 文档完善，API 设计清晰
- 🟢 插件系统灵活
- 🟢 性能优秀，支持大量元素

**劣势**：
- 🔴 协议不明确（NOASSERTION），需要确认商用条款
- 🔴 学习曲线较陡，需要理解架构
- 🔴 相比 Excalidraw 功能更底层

**官网**: https://tldraw.dev/

---

### 3. Drawnix

**简介**：国产开源白板 SaaS，一体化解决方案

**GitHub**: https://github.com/plait-board/drawnix

**核心特点**：
- ✅ **一体化** - 思维导图 + 流程图 + 自由画
- ✅ **国产化** - 中文文档友好
- ✅ **Local First** - 本地优先设计
- ✅ **插件架构** - 支持多种 UI 框架

**技术栈**：
- TypeScript
- 插件架构（支持 Angular、React）
- Plait（自研编辑器框架）

**适用场景**：
- 综合性白板应用
- 中文用户群体
- 企业内部协作工具

**优势**：
- 🟢 功能全面，开箱即用
- 🟢 MIT 协议
- 🟢 中文社区支持

**劣势**：
- 🔴 社区规模较小（1.3 万 Stars）
- 🔴 更新频率相对低
- 🔴 技术栈相对复杂

## 选型建议

### 🏆 推荐：tldraw

**理由**：
1. **SDK 定位** - 专为二次开发设计，不是成品应用
2. **扩展性强** - 插件架构，可以完全定制 UI 和功能
3. **技术先进** - 无限画布、实时协作、高性能
4. **社区活跃** - 4.4 万 Stars，持续更新
5. **适合 AI 集成** - 可以轻松添加 AI 辅助功能

**注意事项**：
- ⚠️ 需要确认商用协议（协议标注为 NOASSERTION）
- ⚠️ 学习成本相对高，需要投入时间理解架构

### 备选方案

#### 如果选择 Excalidraw：
- **优势**：开箱即用，用户体验好，MIT 协议明确
- **劣势**：不是 SDK，二次开发需要 fork 和深度改造
- **适合场景**：想要快速上线，对定制化要求不高

#### 如果选择 Drawnix：
- **优势**：功能全面，中文友好
- **劣势**：社区较小，技术栈复杂
- **适合场景**：需要完整的白板解决方案，面向中文用户

## 🚨 重要补充：素材库对比

**详见**: [白板素材库对比.md](./白板素材库对比.md)

### 核心差异

- **Excalidraw**: 拥有丰富的公共素材库（数千个），社区持续贡献
- **tldraw**: 无内置素材库，仅提供基础形状，需要开发者自建

### 如果素材库是核心需求

**强烈建议重新考虑选型** → **Excalidraw 可能更合适**

原因：
1. 素材库从零构建成本高（1-2 个月）
2. 用户期望有丰富的预制素材
3. Excalidraw 同样可以二次开发和 AI 集成
4. 时间成本和用户价值的权衡
