# AI白板 - 产品概述

**状态**: 🟡 草稿
**创建日期**: 2026-01-15
**版本**: v0.2

---

## 1. 产品定位

AI白板是一个**智能化的在线协作白板工具**，通过融合人工智能能力，为用户提供更高效、更智能的视觉协作体验。

### 1.1 核心理念
- **AI 原生**: 不是在传统白板上简单添加 AI 功能，而是从零开始用 AI 思维构建
- **智能协作**: AI 理解用户意图，主动提供建议和辅助
- **简单易用**: 降低白板工具的学习成本，让创意表达更流畅

---

## 2. 目标用户

### 2.1 主要用户群体（待细化）
- **远程团队**: 需要在线协作的分布式团队
- **创意工作者**: 设计师、产品经理、创业者
- **教育场景**: 在线教学、知识分享
- **个人用户**: 个人笔记、思维整理

### 2.2 用户痛点（待验证）
- 现有白板工具学习成本高
- 缺乏智能化辅助功能
- 协作效率不够高
- 素材资源不够丰富

---

## 3. 核心价值主张

### 3.1 对用户的价值
- **快速上手**: 低学习成本，即开即用
- **AI 辅助**: 智能建议、自动整理、内容生成
- **丰富素材**: 海量图标、图形、模板
- **实时协作**: 流畅的多人协作体验

### 3.2 差异化竞争力
- **完全由 AI 构建**: 开发、设计全流程 AI 参与
- **AI 深度集成**: 不只是功能叠加，而是深度融合
- **待补充**: 具体差异化点需进一步规划

---

## 4. 产品形态

### 4.1 基础形态
- **Web 应用**: 浏览器访问，无需安装
- **实时协作**: 多人同时编辑
- **云端存储**: 自动保存，多端同步

### 4.2 技术基础
- **白板引擎**: Excalidraw（手绘风格，丰富素材库）
- **前端**: React + TypeScript
- **后端**: Node.js
- **架构**: Monorepo (pnpm + Turborepo)

### 4.3 页面结构设计（MVP）

**决策**: 采用**直接白板页面**方案，不设置独立的 Landing Page

**页面布局**:
```
┌─────────────────────────────────────────────────────┐
│ [Logo] AI白板  |  [关于] [示例] [登录/注册]        │ ← 顶部导航栏（固定，高度 ~60px）
├─────────────────────────────────────────────────────┤
│                                                     │
│                                                     │
│                  白板画布区域                        │ ← 核心区域（90%+ 屏幕空间）
│              (Excalidraw 组件)                      │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**设计原则**:
- **极简导航**: 顶部栏占用最小空间，保留必要功能入口
- **打开即用**: 无需登录即可开始绘图（游客模式）
- **首次引导**: 新用户首次访问显示简单 Onboarding（可关闭）
- **渐进式功能**: 核心功能优先，高级功能逐步引入

**路由设计** (MVP 阶段):
- `/` - 主页面（白板）
- `/about` - 关于页面（可选，模态框或简单页面）
- `/board/:id` - 分享的白板（后续支持）

**为什么选择这个方案**:
- ✅ **快速验证**: 专注 AI 白板核心价值，降低 MVP 开发成本
- ✅ **用户体验**: 打开即用，符合工具类产品特性
- ✅ **对标 Excalidraw**: 与开源版保持一致的简洁体验
- ✅ **扩展性**: 后续可以添加 Dashboard 管理多个白板

**后续演进** (成长阶段):
- 添加白板列表页 `/dashboard`
- 添加 Landing Page 用于营销和 SEO
- 支持团队空间和协作管理

## 5. 核心功能规划（初步）

### 5.1 基础白板功能（P0）
基于 Excalidraw 的核心能力：
- [ ] 绘图工具（矩形、圆形、箭头、线条、手绘等）
- [ ] 文本编辑
- [ ] 图片上传
- [ ] 素材库（图标、图形、模板）
- [ ] 图层管理
- [ ] 导出（PNG、SVG、JSON）

### 5.2 协作功能（P0）
- [ ] 实时多人协作
- [ ] 光标跟随
- [ ] 评论与批注
- [ ] 版本历史

### 5.3 AI 功能（核心差异化，优先级待定）
**方向一：AI 辅助创作**
- [ ] AI 生成图形/图标
- [ ] AI 文案建议
- [ ] AI 布局优化
- [ ] AI 配色建议

**方向二：AI 智能识别**
- [ ] 手绘图形识别（转标准图形）
- [ ] 文字识别（图片转文字）
- [ ] 结构识别（自动整理）

**方向三：AI 协作助手**
- [ ] 会议记录整理
- [ ] 思维导图生成
- [ ] 内容总结与提炼

### 5.4 其他功能（P1-P2）
- [ ] 模板市场
- [ ] 团队空间
- [ ] 权限管理
- [ ] 集成第三方工具（Slack、Notion 等）

## 6. 竞品对比

### 6.1 主要竞品
| 产品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| Miro | 专业协作白板 | 功能强大、模板丰富 | 学习成本高、定价贵 |
| FigJam | 轻量协作白板 | 简单易用、Figma 生态 | 功能相对简单 |
| Excalidraw | 开源手绘白板 | 简洁、手绘风格 | 缺少协作、AI 功能 |
| boardmix | 国产白板 | 本地化好 | AI 能力弱 |

详见：[AI白板产品竞品调研](../docs/AI白板产品竞品调研.md)

### 6.2 差异化策略
- **AI 原生**: 不是功能叠加，而是 AI 深度参与
- **手绘风格**: 基于 Excalidraw，视觉友好
- **开源基因**: 基于开源项目，可持续发展
- **待补充**: 需进一步明确差异化

## 9. 关键问题与决策

### 9.1 已确定的决策

✅ **页面结构**:
- **决策**: MVP 阶段采用直接白板页面，不设置 Landing Page
- **日期**: 2026-01-15
- **理由**: 快速验证核心价值，降低开发成本，打开即用
- **影响**: 前端选择轻量级 SPA 方案（Vite + React），暂不需要 SSR

### 9.2 待明确的核心问题

1. **MVP 要包含哪些 AI 功能？**
   - 需要平衡开发成本和差异化价值
   - 需要验证用户需求
   - **优先级**: P0

2. **前端框架最终选择**:
   - Vite + React（轻量，适合 SPA）vs Next.js（功能全，适合后续 SEO）
   - **倾向**: Vite + React（MVP 优先）
   - **优先级**: P0

3. **目标用户是谁？**
   - 个人用户 vs 团队用户
   - 专业用户 vs 普通用户
   - 需要做用户调研
   - **优先级**: P1

4. **如何实现实时协作？**
   - WebSocket + OT/CRDT
   - 自建 vs 第三方服务（如 Liveblocks、y-websocket）
   - **优先级**: P1

5. **AI 能力如何接入？**
   - 模型选择（GPT-4、Claude、开源模型）
   - 成本控制
   - 响应速度
   - **优先级**: P0
