## 📝 文档规范

### PRD 文档命名
- 使用中文描述性名称
- 示例：`用户协作功能需求.md`、`AI辅助绘图PRD.md`

### PRD 文档模板
参考 [PRD模板.md](./PRD模板.md)

### 文档状态标识
在文档开头标注状态：
- 🟡 **草稿** - 初步构思，待讨论
- 🔵 **评审中** - 已完成初稿，待团队评审
- 🟢 **已确认** - 评审通过，待开发
- ✅ **已实现** - 功能已上线
- 🔴 **已废弃** - 不再执行

## 🔄 与技术文档的关系

- **PRDs** (`prds/`): 定义"做什么"、"为什么做"
- **技术文档** (`docs/`): 定义"怎么做"、技术选型
- **ADR** (`.claude/decisions/`): 记录技术决策过程
